# 单股票长期追踪功能设计方案

> 针对个人投资者专注跟踪1-3只股票的实用场景

---

## 一、核心问题

### 1.1 问题描述

```
用户痛点:
今天AI分析: 买入 ⬆️ (理由: 技术面金叉，均线多头)
明天AI分析: 持有 ➡️ (理由: RSI超买，短线回调风险)

问题:
1. 为什么信号变了？是什么因素变了？
2. 这是噪音还是趋势反转？
3. 我应该听今天的还是昨天的？
4. 我的持仓该怎么办？
```

### 1.2 投资经理视角的解读

```
20年经验告诉我们:
- 短期信号频繁变化是正常的，因为市场每天都在消化新信息
- 重要的是理解"变化的原因"而不是"变化本身"
- 需要区分"噪音"和"信号"
- 长期投资者需要"核心逻辑"不变 + "短期因素"动态调整

正确的做法:
1. 建立"核心投资逻辑" - 为什么买入这只股票（3-5年维度）
2. 设定"观察指标" - 哪些信号变化需要关注
3. 设置"预警阈值" - 什么时候需要重新评估
4. 记录"分析历史" - 对比前后分析，理解变化原因
```

### 2.2 功能模块

| 模块 | 功能 | 用户价值 |
|------|------|---------|
| **核心逻辑** | 投资理由/观察指标/止损逻辑 | 明确"为什么买" |

## 四、用户交互流程

### 4.1 开始追踪流程

```

  → 什么情况我会卖？(止损/止盈/逻辑破坏)
  ↓
步骤4: 添加持仓（如有）
  → 买入价格/数量/日期
  ↓
步骤5: 设置预警
  → 基于核心逻辑的智能预警
  → 技术面预警
  → 新闻舆情预警
  ↓
步骤6: 开启追踪
  → 每日自动分析
  → 变化即时通知
```

### 4.2 每日使用流程

  ↓

### 4.3 遇到信号变化的处理流程

```
场景: 昨天买入信号 → 今天持有信号

步骤1: 查看变化原因
  "RSI从58→75，进入超买区域"
  "主力净流入减少"
  ↓
步骤2: 对比核心逻辑
  "核心逻辑中的品牌溢价/ROE有变化吗？" → 无
  "观察指标有恶化吗？" → 无
  ↓
步骤3: 区分噪音与信号
  → 短期技术指标变化 → 噪音，不影响核心逻辑
  → 核心逻辑破坏 → 信号，需要重新评估
  ↓
步骤4: 做出决策
  持仓者: 持有，不因噪音操作
  空仓者: 等待回调至支撑位再买入
  ↓
步骤5: 记录思考
  "今天信号降级是技术面短期调整
   不影响长期投资逻辑
   继续持有，观察明日走势"
```

# 区分噪音与信号
def is_noise_vs_signal(change: Change, thesis: InvestmentThesis) -> str:
    """
    区分噪音和信号
    """
    # 核心逻辑相关 = 信号
    if change.type == 'fundamental' and change.factor in thesis.watchIndicators:
        return 'signal'
    
    # 短期技术指标 = 噪音
    if change.type == 'technical' and change.factor in ['RSI', 'KDJ', '短期均线']:
        return 'noise'
    
    # 量价配合变化 = 需观察
    if change.type == 'volume':
        return 'watch'
    
    return 'neutral'
